# Source Notes

Primary lecture:

- Lecture 3: story structures

Condensed notes:

- Structure is best treated as a diagnostic scaffold.
- Story units need jobs, not just events.
- Try-fail cycles are one of the clearest motion engines.
- Reversals work when they change meaning rather than merely shocking the reader.
- Progress can include backsliding if the slide itself changes the story.
- Nested structures matter: local scenes should support larger arcs.
- If the story feels inert, sharpen decision and consequence before adding more material.

Operational implications:

- When a user asks for outline help, define section jobs first.
- When a story drags, inspect whether attempts actually change the situation.
- When the user wants a twist, verify that a reframe is earned by prior setup.
