# Cases

This folder is for public or internal demonstration cases that prove the harness is doing real work.

Each case should show:

- the target unit
- the benchmark and rubric used
- the before version
- the judged failure
- the intervention
- the after version
- the score delta

## Why Cases Matter

For investors, collaborators, and future users, a before/after case is more convincing than a claim that "the system writes well."

Strong cases make the improvement legible.
