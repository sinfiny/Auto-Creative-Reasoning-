# Method Map

Load this when a task needs the full causal worldbuilding workflow.

## The Engine

The core loop is:

```text
Want -> Obstacle -> Action -> Consequence -> New pressure
```

The loop can run forward to generate lore or backward to explain existing lore. It should be recursive: a consequence becomes a new context that creates the next want, obstacle, and action.

## Five Forces

Use these as active pressures, not wiki categories:

- Environment: geography, climate, resources, body constraints, magic geography, natural threats
- Power: authority, hierarchy, control, enforcement, legitimacy, dependency
- Belief: religion, philosophy, values, taboo, cosmology, moral explanation
- Exchange: trade, conflict, borrowing, diplomacy, interdependence, isolation
- Legacy: history, trauma, inherited debt, conquest, migration, collective memory

One force's consequence can become another force's obstacle. Example pattern:

```text
Environment creates water scarcity.
Scarcity creates control points.
Control points create authority.
Authority needs legitimacy.
Legitimacy creates sacred water belief.
Sacred water belief creates art, taboo, ritual, and status.
```

## Base-Layer Inputs

### Landscape

- Movement cost: how hard distance is
- Visibility: how easily threats, wealth, and weakness can be seen
- Defensibility: how easily a place, resource, route, or body can be protected
- Connectivity: how much contact with outsiders happens by default

### Climate

- Predictability: whether people can plan around seasonal patterns
- Survival stress: how expensive failure is
- Resource cycling: whether essential resources are stored, renewed, followed, traded, or transformed into favors

### Resources

- Scarcity: what gap must ingenuity, migration, conquest, ritual, or trade solve?
- Abundance: what surplus creates specialization, envy, stewardship, taboo, or invasion?
- Storage: can wealth be accumulated?
- Portability: can wealth be hidden, moved, stolen, traded, or taxed?
- Control points: who can block access?

### Threats

- Predictable, frequent, defensible threats tend to create practical institutions.
- Unpredictable or undefendable threats tend to create explanation systems, ritual control, priesthoods, omens, taboos, or fatalism.

## Six-Layer Stack

Generate in order:

1. Survival strategy and material culture
2. Social organization and communication
3. Values, taboos, virtues, shame, and honor
4. Worldview, religion, time, death, and metaphysics
5. Symbolic culture: architecture, dress, art, music, tools, titles
6. Daily expressions: idioms, greetings, etiquette, food, reflexes, rituals

Layer rule:

- A layer is valid only if it protects, explains, legitimizes, or expresses a lower-layer pressure.
- If a layer does not have a lower-layer cause, mark it as decorative and reverse-engineer it.

## Reverse Workflow

Use this for a trait that already exists.

```text
Existing element:
Immediate function:
Action that could create it:
Obstacle that required the action:
Want behind the obstacle:
Bedrock:
If true, what else must be true:
Five-force gaps:
Values produced:
Contradictions produced:
```

Shallow reverse:

- Go back one cycle, then ripple forward to create connected details.

Deep reverse:

- Keep asking why until reaching geography, body, magic, historical event, or human universal.

## Culture Audit

Use these checks after generation:

- Action audit: What behaviors appear repeatedly?
- Obstacle audit: What problems recur?
- Consequence audit: What became permanent?
- Force audit: Is environment, power, belief, exchange, and legacy represented?
- Stack audit: Does each cultural layer depend on the one beneath it?
- Friction audit: Which values contradict each other in useful ways?

## Species and Biology

Treat biology as an environment that travels with the people.

Useful biological variables:

- Sensory range
- Locomotion
- Lifespan
- Reproduction rate
- Sleep requirements
- Diet
- Pain response
- Memory
- Communication mode
- Body maintenance needs
- Magic produced by the body


## Politics and Power

Political systems are built from power centers and currencies of power.

Ask:

- Who controls what?
- Why does everyone else accept it?
- What visible currency do people fight over?
- What hidden enforcement makes the visible currency matter?
- Which middle layers have their own incentives?

Power-center loop:

```text
Power center:
Currency:
Want:
Obstacle:
Action:
Consequence:
Who is destabilized:
What new leverage appears:
```

Middle-layer rule:

- Avoid flat "king plus great houses" politics.
- Add vassals, clients, provinces, guild chapters, bureaucrats, officers, companies, or local brokers.
- Give every middle layer a loyalty reason, a breaking point, and at least one cross-connection.

## Maps

Maps are tools, not starting points.

Use a map when geography creates decisions or constraints:

- Journey pressure
- Travel time
- Rations and supply
- Trade routes
- borders
- strategic chokepoints
- political territory
- continuity tracking

Map audit:

```text
Story/campaign need:
Key locations:
Travel-time relationships:
Obstacles between them:
Who controls routes:
Which decisions the map creates:
What changes if the map is removed:
```
