# Skill Lens Map

Use this map when composing a benchmark or choosing which repository skills should act as evaluative lenses.

## Core Control

- `creative-autoresearch-control-plane`
  - use for loop control, failure routing, and keep/discard decisions
- `autonomous-webserial-loop`
  - use for strict strategic-webserial chapter iteration

## Structure And Strategy

- `strategic-webserial-architecture`
  - scores promise stacks, clue ladders, reveal order, and irreversible state change
- `strategic-cast-intelligence-engine`
  - scores actor intelligence, leverage use, countermoves, and believable mistakes
- `faction-power-consequence-engine`
  - scores system cost, institutional pressure, exploit space, and consequence chains
- `mystery-payoff-and-fairness-judge`
  - scores fairness, causal integrity, consequence density, and payoff inevitability

## Scene And Chapter Craft

- `narrative-hook-hold-payoff`
  - scores entry pressure, escalation, payoff, and changed story state
- `narrative-explicit-reasoning`
  - scores coherence of motive, lore, assumptions, and contradiction handling
- `narrative-auto-reasoning`
  - coordinates multi-skill scene/chapter work when more than one control loop is needed

## Information And Prose

- `narrative-epistemic-targeting`
  - scores how well the draft controls reader knowledge and reveal timing
- `narrative-subtext-mapping`
  - scores hidden agendas, emotional state, and dialogue subtext
- `narrative-contextual-prose-density`
  - scores detail budget and depiction versus explanation balance
- `narrative-minimum-viable-prose`
  - scores compression, anti-bloat discipline, and concrete emotional delivery
- `narrative-surgical-editing`
  - use when the benchmark implies a constrained local fix

## Worldbuilding

- `grainbound-causal-worldbuilding`
  - scores whether lore, culture, and institutions emerge from causal pressures instead of vibes

## Benchmark Composition Rule

Do not select every lens.

Choose the smallest set of skills that can judge the actual promise of the fiction unit under test.

A good custom benchmark usually has:

- one control skill
- two to four scoring lenses
- a clear pass floor
- an explicit rewrite routing priority
