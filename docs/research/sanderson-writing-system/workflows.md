# Workflow Catalog

These are the workflows that can be made from the lecture series. They are written so an AI or a student can either run a single slice or follow the whole methodology.

## 1. Writer Process Calibration

Use when the problem is inconsistency, endless restarting, or confusion about outlining vs discovery.

Steps:

1. Name the current writing habit and current failure mode.
2. Place the writer on the architect-to-gardener spectrum for this project.
3. Decide what must be discovered in draft and what must be known in advance.
4. Set a finishing-first rule for the current manuscript.
5. Define a revision point only after a full draft or a major segment is complete.

Primary skill:

- `$sanderson-professional-writing`

## 2. Promise Audit

Use when an opening is weak, a middle drifts, or an ending feels like it belongs to another book.

Steps:

1. List the active tone, plot, character, genre, and worldbuilding promises.
2. Mark where each promise is first made.
3. Check whether the middle shows visible progress on each promise.
4. Mark which promises are intentionally broken, transformed, or paid off.
5. Repair missing signposts or mismatched payoffs.

Primary skill:

- `$sanderson-promise-progress-payoff`

## 3. Story Architecture Loop

Use when the book feels shapeless, repetitive, or low on escalation.

Steps:

1. Define the job of each act or major sequence.
2. Identify current desire, obstacle, and cost.
3. Build a try-fail or try-succeed-with-cost cycle.
4. Add a reversal, reveal, or commitment point where needed.
5. Verify that each section changes the story state.

Primary skill:

- `$sanderson-story-architecture`

## 4. Character Engine Design

Use when characters feel passive, forgettable, or overdesigned without producing investment.

Steps:

1. Define what the reader should feel toward the character.
2. Tune the lever mix: proactivity, capability, relatability, sympathy, fascination.
3. Decide the arc type.
4. Define an early inability, refusal, wound, or pressure point.
5. Define the late action that proves change, refusal, or consequence.

Primary skill:

- `$sanderson-character-engine`

## 5. Magic System Design

Use when building a speculative system or fixing a climax that feels unfair.

Steps:

1. Choose desired emotional effect: wonder, dread, puzzle, tactical satisfaction, mythic scale.
2. Place the system on the hard-soft spectrum.
3. Define costs, limits, and reader-legible rules at the needed level.
4. Seed tools or patterns early.
5. Make the climax about character choice using the system, not just about system cleverness.

Primary skill:

- `$sanderson-speculative-design`

## 6. Narrative Worldbuilding Filter

Use when worldbuilding is sprawling, static, or detached from story.

Steps:

1. List the setting elements currently in play.
2. Keep the ones that affect choice, cost, status, danger, or social pressure.
3. Cut or defer encyclopedia material that does not alter behavior.
4. Reintroduce essential setting through action, viewpoint, and consequence.
5. Time-box future worldbuilding and return to drafting.

Primary skill:

- `$sanderson-speculative-design`

## 7. Earth Setting Relevance Pass

Use when the story uses a real-world setting but the material reads like a travel note or research dump.

Steps:

1. Decide what the viewpoint character notices and ignores.
2. Tie setting facts to tension, identity, or consequence.
3. Replace neutral trivia with social, political, economic, or emotional consequence.
4. Check with outside readers for missing context.
5. Keep only the location detail that changes how scenes play.

Primary skill:

- `$sanderson-speculative-design`

## 8. Ending Design Loop

Use when the ending feels clever-but-empty, tragic-but-cheap, or mechanically solved.

Steps:

1. Restate the main promises.
2. Identify which promise is central and must land emotionally.
3. Trace seeded tools, flaws, relationships, and costs.
4. Design a final choice that reveals who the protagonist has become.
5. Verify that the resolution pays the promised currency.

Primary skill:

- `$sanderson-endings-and-revision`

## 9. Revision Experiment Ladder

Use when the manuscript is complete enough for revision but the next pass is unclear.

Steps:

1. Gather high-signal feedback from alpha or beta readers.
2. Sort signals into recurring friction points.
3. Decide the likely underlying causes rather than obeying every suggested fix.
4. Run one deliberate pass at a time: structure, promise, character, prose compression, or pacing.
5. Re-test the revised manuscript and compare results.

Primary skill:

- `$sanderson-endings-and-revision`

## 10. Agent and Publishing Path Selection

Use when deciding between traditional, indie, or hybrid strategies.

Steps:

1. Clarify the goal: speed, control, prestige, distribution, rights leverage, income shape.
2. Decide whether the project is novel, novella, serial, or side format.
3. Research the relevant market and current operator set.
4. Build the path-specific asset list: agent targets, mailing list, backlist, platform, cover, ad budget, or conventions.
5. Choose one path for the current project and stop mixing conflicting tactics.

Primary skill:

- `$sanderson-publishing-strategy`

## 11. Mailing List and Platform Workflow

Use when the author needs audience building without losing all writing time to social media.

Steps:

1. Decide whether the author has a real platform angle or only book updates.
2. Default to a mailing list if platform value is unclear.
3. Define what value the newsletter provides.
4. Set a sustainable cadence.
5. Keep social channels minimal unless they clearly help the career.

Primary skill:

- `$sanderson-publishing-strategy`

## 12. Full Sanderson Methodology

Use when a project needs the whole system.

Steps:

1. Calibrate process and constraints.
2. Define premise and promises.
3. Build the structure and progress engine.
4. Build the character engine.
5. Build the speculative design and worldbuilding filter.
6. Draft toward a character-centered payoff.
7. Run revision experiments.
8. Choose a publishing path only after the manuscript strategy is clear.

Primary skill:

- `$sanderson-full-methodology`
