# Untracked Integration Plan

This repository currently contains a meaningful amount of local work that is not yet tracked on `main`.

That is normal for an active research workspace, but it creates two risks:

- important insights stay invisible to collaborators and investors
- future sessions may rebuild work that already exists locally

## Triage Buckets

Every untracked folder or report should be sorted into one of these buckets.

### 1. Commit Soon

Use this bucket for:

- finished or nearly finished skills
- reusable reports that inform benchmarks or product strategy
- reference material that future agents should actually see

### 2. Keep Local For Now

Use this bucket for:

- rough experiments
- half-formed skill ideas
- reports that still need cleanup
- prompt or harness variants that are not yet stable

### 3. Ignore Permanently

Use this bucket for:

- raw caches
- downloaded media
- node modules
- temporary scrape dumps
- machine-specific clutter

## Recommended Near-Term Integration Order

1. `docs/research/fanfiction-platform-data/`

## Integration Rule

Do not merge untracked work in bulk.

Instead:

1. choose one family
2. clean the folder
3. add or update README references if needed
4. commit only that family
5. repeat

## Why This Matters For YC

For a repo like this, the proof is not just in private local progress. The proof is in visible, coherent artifacts:

- skills
- benchmark docs
- evaluation rubrics
- public demo cases
- disciplined commits that show a system becoming legible
